# LLM Chatbot 프로젝트 개요

## 한 줄 요약

사내 메신저 Cube에 연동되는 **OpenAI 호환 로컬 LLM 기반 AI 챗봇 서버**로, 사내 인프라 안에서 대화 처리와 CDN 파일 제공을 함께 담당합니다.

---

## 배경 및 목적

### 문제 인식

- 업무 중 반복적인 데이터 조회, 시각화 요청이 발생하지만, 이를 위해 별도 시스템에 접속하거나 담당자에게 요청해야 하는 비효율 존재
- 외부 AI 서비스(OpenAI, Anthropic 등)는 보안 정책상 사내 데이터를 전송할 수 없음

### 해결 방향

- **사내 로컬 LLM**을 활용하여 보안 이슈 없이 AI 기능을 제공
- 이미 사용 중인 **Cube 메신저**에 통합하여 별도 앱 설치 없이 자연어로 즉시 이용 가능
- 대화 이력과 운영 로그를 서버 쪽에서 관리하여 일관된 응답 흐름을 유지

---

## 주요 기능

| 기능 | 설명 |
|------|------|
| **자연어 대화** | 한국어 질문에 대해 LLM이 답변 생성 |
| **대화 기록 유지** | 최근 대화 맥락을 유지하여 이전 대화를 이어갈 수 있음 |
| **비동기 메시지 처리** | Cube 수신 요청은 큐에 적재하고 worker가 실제 LLM 처리를 수행 |
| **Cube 메신저 연동** | Cube에서 메시지를 받아 처리 후 `multiMessage`로 응답 |
| **CDN 파일 처리** | 파일 업로드, 원본 다운로드, 이미지 리사이즈/썸네일 제공 |
| **운영 로그/스케줄러** | 구조화 activity 로그와 APScheduler 기반 백그라운드 작업 지원 |

---

## 기술 스택

| 구분 | 기술 |
|------|------|
| **백엔드** | Python, Flask, uWSGI |
| **LLM** | OpenAI 호환 LLM API |
| **메시지 큐/일부 상태 저장** | Redis |
| **대화 저장** | Redis (현재) / In-Memory fallback |
| **로그 저장** | 파일 기반 JSONL + stdout |
| **메신저 연동** | Cube API |
| **스케줄러** | APScheduler |

## 시스템 구성

```
사용자
  │
  ▼
Cube 메신저
  │ POST /api/v1/cube/receiver
  ▼
LLM Chatbot 서버 (Flask + uWSGI)
  ├─ cube/router.py
  ├─ cube/service.py
  ├─ cube/queue.py
  ├─ cube/worker.py
  ├─ conversation_service.py
  ├─ llm/service.py
  ├─ cdn/cdn_service.py
  └─ utils/logger/, utils/scheduler/
      │
      ├─ Redis
      │   - Cube 수신 큐
      │   - 대화 이력(현재 구현)
      │   - CDN 메타데이터
      │   - 스케줄러 락
      │
      ├─ 파일시스템/PVC
      │   - CDN 파일
      │   - 로그 파일
      │
      └─ OpenAI 호환 LLM 서버
```

## 핵심 설계 원칙

1. **외부 AI 서비스 의존 제로** — 모든 LLM 호출은 사내 로컬 서버로만 전송
2. **비동기 응답 처리** — 사용자 요청을 즉시 큐에 적재하고 worker가 백그라운드에서 LLM 처리 후 Cube로 결과 전달
3. **환경 변수 기반 설정** — 모든 URL, 인증 정보, 모델명은 환경 변수로 관리하여 환경별 전환 용이
4. **단순한 저장 경로 유지** — 현재는 Redis 기반 최근 대화 이력 저장을 사용하고, 향후 MongoDB 원문 저장 구조로 확장 가능하게 설계

---

## 기대 효과

- **업무 효율화** — Cube 안에서 자연어로 빠르게 질의응답 가능
- **보안 준수** — 사내 데이터가 외부로 유출되지 않음
- **접근성 향상** — 별도 도구 없이 기존 Cube 메신저에서 바로 사용
- **확장 가능** — 대화 저장소 고도화, summary compaction, 검색 계층 추가로 점진 확장 가능
