# LLM Chatbot 프로젝트 개요

## 한 줄 요약

사내 메신저 Cube에 연동되는 **로컬 LLM 기반 AI 챗봇**으로, 외부 클라우드 AI 서비스 없이 사내 인프라만으로 동작합니다.

---

## 배경 및 목적

### 문제 인식

- 업무 중 반복적인 데이터 조회, 시각화 요청이 발생하지만, 이를 위해 별도 시스템에 접속하거나 담당자에게 요청해야 하는 비효율 존재
- 외부 AI 서비스(OpenAI, Anthropic 등)는 보안 정책상 사내 데이터를 전송할 수 없음

### 해결 방향

- **사내 로컬 LLM**을 활용하여 보안 이슈 없이 AI 기능을 제공
- 이미 사용 중인 **Cube 메신저**에 통합하여 별도 앱 설치 없이 자연어로 즉시 이용 가능
- LLM이 직접 **도구(Tool)**를 호출하여 데이터 조회·차트 생성까지 자동 수행

---

## 주요 기능

| 기능 | 설명 |
|------|------|
| **자연어 대화** | 한국어 질문에 대해 LLM이 답변 생성 |
| **데이터 조회** | LLM이 Tool Use를 통해 데이터 소스에서 직접 정보를 조회 |
| **차트 생성** | 조회된 데이터를 기반으로 bar, line, pie 차트를 자동 생성하여 이미지로 전달 |
| **대화 기록 유지** | 사용자별 대화 맥락을 유지하여 이전 대화를 이어갈 수 있음 |
| **Cube 메신저 연동** | Cube에서 메시지를 받아 처리 후 결과를 리치 알림(텍스트 + 이미지)으로 응답 |

---

## 기술 스택

| 구분 | 기술 |
|------|------|
| **백엔드** | Python, Flask, uWSGI |
| **LLM** | 로컬 LLM (Kimi-K2.5, Qwen3, GPT-OSS) — OpenAI 호환 API 형식 |
| **대화 저장** | Redis (운영) / In-Memory (개발) |
| **로그 저장** | MongoDB |
| **차트 생성** | Matplotlib |
| **메신저 연동** | Cube API |

## 핵심 설계 원칙

1. **외부 AI 서비스 의존 제로** — 모든 LLM 호출은 사내 로컬 서버로만 전송
2. **비동기 응답 처리** — 사용자 요청을 즉시 수락(202)하고 백그라운드에서 LLM 처리 후 Cube로 결과 전달
3. **환경 변수 기반 설정** — 모든 URL, 인증 정보, 모델명은 환경 변수로 관리하여 환경별 전환 용이
4. **Tool Use 아키텍처** — LLM이 필요에 따라 자율적으로 도구를 선택·호출하여 복합 작업 수행 (최대 5회 반복)

---

## 기대 효과

- **업무 효율화** — 자연어 한 문장으로 데이터 조회 + 시각화까지 완료
- **보안 준수** — 사내 데이터가 외부로 유출되지 않음
- **접근성 향상** — 별도 도구 없이 기존 Cube 메신저에서 바로 사용
- **확장 가능** — Tool 추가만으로 새로운 기능(리포트 생성, 시스템 조회 등)을 쉽게 확장
