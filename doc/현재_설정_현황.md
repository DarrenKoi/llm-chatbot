# 현재 설정 현황

이 문서는 프로젝트의 런타임 구성, 환경 변수, 프로세스 구조, 저장소 백엔드, 워크플로 목록 등 **현재 운영 설정**을 한곳에 정리한다.

> 최종 갱신: 2026-04-04

---

## 1. 프로세스 구성

uWSGI가 메인 프로세스이며, `attach-daemon`으로 두 워커를 함께 띄운다.

| 프로세스 | 진입점 | 역할 |
|----------|--------|------|
| **Flask App** | `index.py` → `api.create_application()` | HTTP 요청 처리 |
| **Cube Worker** | `cube_worker.py` → `api.cube.worker.main()` | Redis 큐에서 Cube 메시지를 소비하여 LLM 워크플로 실행 |
| **Scheduler Worker** | `scheduler_worker.py` → `api.scheduler_worker.main()` | APScheduler 기반 정기 작업 실행 |

### uWSGI 주요 튜닝 (`wsgi.ini`)

| 항목 | 값 | 비고 |
|------|----|------|
| workers | 2 | 4 vCPU / 8 GiB 기준 |
| threads | 4 | CDN/HTTP I/O 동시성 |
| harakiri | 120s | 요청 타임아웃 |
| max-requests | 1500 (delta 100) | 메모리 누수 방지 재시작 |
| reload-on-rss | 700 MB | RSS 초과 시 워커 재시작 |
| touch-reload | `/project/workSpace/restart.txt` | 파일 터치로 무중단 재시작 |

## 2. 환경 변수 설정

설정 파일: `api/config.py` (`.env` → `.env.example` 순서로 로드)

### 2.1 Cube 연동

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `CUBE_API_ID` | (필수) | Cube API 인증 ID |
| `CUBE_API_TOKEN` | (필수) | Cube API 인증 토큰 |
| `CUBE_API_URL` | `http://cube.skhynix.com:8888` | Cube 서버 주소 |
| `CUBE_BOT_NAME` | `ITC OSS` | 봇 표시 이름 |
| `CUBE_BOT_USERNAMES` | `ITC OSS` | 봇 사용자명 (쉼표 구분 가능) |
| `CUBE_TIMEOUT_SECONDS` | 10 | Cube API 호출 타임아웃 |

### 2.2 LLM

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `LLM_BASE_URL` | (필수) | OpenAI 호환 LLM 서버 주소 |
| `LLM_API_KEY` | (필수) | LLM API 키 |
| `LLM_MODEL` | (필수) | 사용 모델명 |
| `LLM_SYSTEM_PROMPT_OVERRIDE` | (빈 문자열) | 시스템 프롬프트 오버라이드 |
| `LLM_TIMEZONE` | `Asia/Seoul` | 프롬프트에 주입되는 시간대 |
| `LLM_TIMEOUT_SECONDS` | 30 | LLM 호출 타임아웃 |
| `LLM_THINKING_MESSAGE` | `잠시만요, 답변을 준비하고 있어요... 🤔` | 응답 대기 안내 문구 |
| `LLM_THINKING_MESSAGE_DELAY_SECONDS` | 5.0 | 대기 안내 전송까지 지연 시간 |

**사용 가능 모델**: Kimi-K2.5 (기본), Qwen3, GPT-OSS

## 3. 저장소 백엔드 및 폴백 전략

| 서비스 | 운영 백엔드 | 폴백 (로컬/미설정 시) |
|--------|------------|----------------------|
| **대화 이력** | MongoDB (`AFM_MONGO_URI`) | `OrderedDict` 기반 in-memory (최대 1000 사용자) |
| **메시지 큐** | Redis (`REDIS_URL`) | in-memory 폴백 |
| **스케줄러 분산 락** | Redis (`SCHEDULER_REDIS_URL`) | 락 없이 단독 실행 |
| **파일 전달 메타데이터** | Redis (`FILE_DELIVERY_REDIS_URL`) | in-memory 폴백 |
| **워크플로 상태** | 파일 시스템 (`WORKFLOW_STATE_DIR`) | 항상 파일 시스템 |
| **MCP 캐시** | 파일 시스템 (`MCP_CACHE_DIR`) | 항상 파일 시스템 |
| **대화 아카이빙** | OpenSearch | (미설정 시 비활성) |
| **LangGraph 체크포인터** | MongoDB (`langgraph-checkpoint-mongodb`) | (미설정 시 비활성) |

## 4. 등록된 워크플로

`api/workflows/` 하위에 자동 발견되며, 각 패키지는 `get_workflow_definition()` 또는 `WORKFLOW_DEFINITION` dict를 내보내야 한다.

| 워크플로 ID | 패키지 | 용도 |
|-------------|--------|------|
| `start_chat` | `api/workflows/start_chat/` | 기본 진입 워크플로 (분류 → casual 직접 처리 or handoff) |
| `common` | `api/workflows/common/` | 공통 처리 흐름 |
| `chart_maker` | `api/workflows/chart_maker/` | 차트 생성 |
| `ppt_maker` | `api/workflows/ppt_maker/` | PPT 생성 |
| `recipe_requests` | `api/workflows/recipe_requests/` | 레시피 요청 처리 |
| `at_wafer_quota` | `api/workflows/at_wafer_quota/` | 도메인 특화 질의 처리 |
| `translator` | `api/workflows/translator/` | 번역 워크플로 |
| `travel_planner` | `api/workflows/travel_planner/` | 여행 계획 워크플로 |

## 5. 스케줄러 작업

`api/scheduled_tasks/tasks/` 하위에 자동 발견된다.

| 모듈 | 설명 |
|------|------|
| `cleanup.py` | 오래된 로그/임시 파일 정리 |
| `_examples.py` | cron/interval/date 패턴 레퍼런스 (실행 안 됨, `_` 접두사) |

추가 패키지는 `_registry.py`의 `_TASK_PACKAGES` 리스트에 등록한다.

## 6. LLM 레지스트리

`api/llm/registry.py`에서 관리한다.

- **기본 모델**: `Kimi-K2.5` (`temperature=0.0`)
- 워크플로/노드별 모델 분기 가능 (현재는 기본값 반환)
- LLM 클라이언트는 `langchain-openai`의 `ChatOpenAI` (OpenAI 호환 엔드포인트)

## 7. MCP (Model Context Protocol)

| 구성요소 | 파일 | 설명 |
|----------|------|------|
| 서버 등록 | `api/mcp/registry.py` | `MCPServerConfig` 기반 서버 등록/조회 |
| 도구 등록 | `api/mcp/registry.py` | `MCPTool` 기반 도구 메타데이터 등록 |
| 도구 실행 | `api/mcp/executor.py` | 도구 호출 실행 흐름 |
| 도구 선택 | `api/mcp/tool_selector.py` | LLM이 호출할 도구 결정 |
| 캐시 | `api/mcp/cache.py` | 파일 기반 캐시 (`MCP_CACHE_DIR`) |

---

## 8. Blueprint 자동 로드

`api/blueprint_loader.py`가 `api/` 하위의 `router.py` 또는 `router_*.py` 파일을 재귀 탐색하여 Flask Blueprint를 자동 등록한다.

현재 등록된 라우터:
- `api/cube/router.py` — Cube 메시지 수신 (`/api/v1/cube/receiver`)
- `api/file_delivery/router.py` — 파일 업로드/다운로드
