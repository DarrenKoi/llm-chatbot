# 임베딩 기반 도구 선택 전략 설계

**일시**: 2026-03-23 12:57
**주제**: MCP 도구 토큰 절약을 위한 임베딩 기반 Tool Selection 전략 설계 및 아키텍처 문서 반영

---

## 1. 진행 사항

- **도구 토큰 비용 문제 분석**: LLM에 모든 MCP 도구 정의를 매 요청마다 전달하면 도구 수에 비례하여 context 토큰이 소비되는 문제를 논의
- **3가지 도구 선택 패턴 비교 검토**:
  - Pattern 1: Two-Stage Tool Selection (Router LLM) — 소형 LLM으로 관련 서버 사전 선택
  - Pattern 2: Embedding 기반 도구 선택 — RAG 임베딩 인프라 재사용 (채택)
  - Pattern 3: Hierarchical Agents — MCP 서버별 독립 LLM (오버엔지니어링으로 보류)
- **도구 필요 여부 판단 단계 설계**: 도구 선택 이전에 "이 메시지가 도구를 필요로 하는가?" 판단 단계를 추가하는 아이디어 논의 → 임베딩 유사도 기반으로 두 판단을 하나의 연산으로 통합
- **`doc/RAG_MCP_아키텍처.md`에 §5.10 "도구 선택 전략" 섹션 작성** (11개 하위 섹션)
