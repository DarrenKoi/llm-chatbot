# 사용자 프로필 프롬프트 주입 계획

## 목적

대화가 시작될 때 LLM이 아래 문맥을 먼저 알 수 있게 하는 것이 목표다.

- 오늘 기준 시각
- 사용자 기본 정보: 이름, 소속 팀, 근무지, 직무 등
- 프로필 출처: 상세 URL 또는 Redis fallback

이 문맥은 두 군데에 직접 도움이 된다.

- `start_chat`에서 어떤 workflow로 넘길지 판단할 때
- 일반 답변을 만들 때 사용자 상황에 맞는 표현과 절차를 고를 때

이 문서는 구현 전 검토용 설계안과 샘플 코드만 담는다.

## 현재 코드 기준 삽입 지점

현재 흐름은 대략 아래와 같다.

```text
Cube 메시지 수신
-> orchestrator
-> start_chat.entry
-> classify_intent
-> retrieve_context
-> plan_response
-> generate_reply
-> LLM 호출
```

프로필은 `start_chat.entry` 바로 다음 또는 `classify_intent` 전에 한 번만 불러오는 구성이 가장 단순하다.

권장 이유:

- 첫 턴에서 workflow 라우팅에 바로 쓸 수 있다.
- 같은 턴 안에서 여러 번 URL/Redis를 조회하지 않아도 된다.
- 이후 노드에서는 정규화된 상태값만 읽으면 된다.

## 권장 동작 순서

1. `entry_node()`에서 `user_id` 기준으로 프로필 로딩을 시도한다.
2. 상세 프로필 URL을 짧은 timeout으로 먼저 호출한다.
3. 아래 경우에는 바로 Redis hash fallback으로 내려간다.
   - connect/read timeout
   - DNS/네트워크 오류
   - HTTP 5xx
   - 응답 body 파싱 실패
   - 필수 필드 누락
4. 두 소스의 응답을 공통 `UserProfile` 형태로 정규화한다.
5. `state.data`와 workflow 전용 상태에 아래 값을 저장한다.
   - `profile_summary`
   - `profile_source`
   - `profile_loaded`
6. 라우팅과 프롬프트 생성 시 이 정규화된 값을 사용한다.
7. 프로필을 못 가져와도 workflow는 실패시키지 말고, 시간 문맥만 넣어서 계속 진행한다.

## 실패 처리 기준

상세 URL은 "가용하면 우선 사용, 실패해도 전체 대화는 계속"이라는 기준으로 다루는 편이 안전하다.

- URL timeout은 짧게 잡는다. 예: `1.0 ~ 1.5초`
- Redis fallback은 상대적으로 안정적인 경로로 본다.
- 두 소스가 모두 실패하면 `profile_source="unavailable"`로 저장하고 진행한다.
- 프롬프트에는 없는 필드를 억지로 넣지 않는다.

## 권장 상태 구조

`StartChatWorkflowState`에 아래 필드를 추가하는 형태가 무난하다.

```python
from dataclasses import dataclass, field

from api.workflows.models import WorkflowState


@dataclass
class StartChatWorkflowState(WorkflowState):
    detected_intent: str = "start_chat"
    retrieved_contexts: list[str] = field(default_factory=list)

    profile_loaded: bool = False
    profile_source: str = ""
    profile_summary: str = ""
```

## 권장 프롬프트 전략

프로필은 시스템 프롬프트 뒤에 "추가 실행 문맥"으로 붙이는 편이 가장 단순하다.

문맥 구성 예시:

```text
현재 한국 현지 시각은 2026-04-04 10:15:00 (Asia/Seoul, UTC+09:00)입니다.
오늘, 내일, 이번 주 같은 상대적 시간 표현은 이 시각을 기준으로 해석하세요.

[사용자 프로필]
- 이름: 홍길동
- 소속: ITC OSS / Platform Engineering
- 근무지: 이천
- 직무: MLOps
- 프로필 출처: profile_api

이 사용자의 소속과 근무 맥락을 고려해 적절한 workflow 또는 답변 방식을 선택하세요.
```

핵심은 아래 두 가지다.

- 시간 문맥과 사용자 문맥을 한 곳에서 조립한다.
- LLM에 넘기는 정보는 정규화된 짧은 요약 문자열로 제한한다.

## 추천 파일 구성

구현 시에는 아래 정도로 나누면 책임이 분명하다.

- `api/profile/service.py`: URL 우선, Redis fallback, 정규화
- `api/profile/models.py`: `UserProfile` dataclass
- `api/workflows/start_chat/state.py`: 프로필 상태 저장
- `api/workflows/start_chat/nodes.py`: 첫 턴에서 프로필 선조회
- `api/llm/prompt/system.py`: 시간 + 프로필 문맥 조립
- `api/llm/service.py`: `extra_system_context` 같은 확장 인자 허용

### 2. 프로필 모델 예시

`api/profile/models.py`

```python
from dataclasses import dataclass


@dataclass
class UserProfile:
    user_id: str
    name: str = ""
    team: str = ""
    organization: str = ""
    work_location: str = ""
    role: str = ""
    email: str = ""
    source: str = ""

    def to_prompt_text(self) -> str:
        lines: list[str] = []
        if self.name:
            lines.append(f"- 이름: {self.name}")
        if self.organization or self.team:
            group_name = " / ".join(part for part in [self.organization, self.team] if part)
            lines.append(f"- 소속: {group_name}")
        if self.work_location:
            lines.append(f"- 근무지: {self.work_location}")
        if self.role:
            lines.append(f"- 직무: {self.role}")
        if self.source:
            lines.append(f"- 프로필 출처: {self.source}")
        return "\n".join(lines)
```

### 3. URL 우선 + Redis fallback 서비스 예시

`api/profile/service.py`

```python
import logging
from typing import Any

import httpx

from api import config
from api.profile.models import UserProfile

log = logging.getLogger(__name__)


class UserProfileLookupError(RuntimeError):
    """프로필 조회가 모두 실패했을 때 사용한다."""


def load_user_profile(user_id: str) -> UserProfile | None:
    profile = _load_from_profile_api(user_id)
    if profile is not None:
        return profile

    profile = _load_from_redis(user_id)
    if profile is not None:
        return profile

    return None


def _load_from_profile_api(user_id: str) -> UserProfile | None:
    template = config.USER_PROFILE_API_URL.strip()
    if not template:
        return None

    url = template.format(user_id=user_id)

    try:
        response = httpx.get(url, timeout=config.USER_PROFILE_API_TIMEOUT_SECONDS)
        response.raise_for_status()
        payload = response.json()
    except (httpx.RequestError, httpx.HTTPStatusError, ValueError) as exc:
        log.warning("profile_api_lookup_failed user_id=%s error=%s", user_id, exc)
        return None

    if not isinstance(payload, dict):
        log.warning("profile_api_lookup_invalid_payload user_id=%s", user_id)
        return None

    return _normalize_profile(payload, user_id=user_id, source="profile_api")


def _load_from_redis(user_id: str) -> UserProfile | None:
    redis_url = config.USER_PROFILE_REDIS_URL.strip()
    if not redis_url:
        return None

    key = f"{config.USER_PROFILE_REDIS_KEY_PREFIX}:{user_id}"

    try:
        import redis

        client = redis.from_url(redis_url, socket_connect_timeout=1, socket_timeout=1)
        raw = client.hgetall(key)
    except Exception as exc:
        log.warning("profile_redis_lookup_failed user_id=%s error=%s", user_id, exc)
        return None

    if not raw:
        return None

    decoded = {
        _decode_redis_value(field): _decode_redis_value(value)
        for field, value in raw.items()
    }
    return _normalize_profile(decoded, user_id=user_id, source="redis")


def _normalize_profile(payload: dict[str, Any], *, user_id: str, source: str) -> UserProfile | None:
    name = str(payload.get("name") or payload.get("user_name") or "").strip()
    team = str(payload.get("team") or payload.get("team_name") or "").strip()
    organization = str(payload.get("organization") or payload.get("org_name") or "").strip()
    work_location = str(payload.get("work_location") or payload.get("site") or "").strip()
    role = str(payload.get("role") or payload.get("job_title") or "").strip()
    email = str(payload.get("email") or "").strip()

    if not any([name, team, organization, work_location, role, email]):
        return None

    return UserProfile(
        user_id=user_id,
        name=name,
        team=team,
        organization=organization,
        work_location=work_location,
        role=role,
        email=email,
        source=source,
    )


def _decode_redis_value(value: bytes | str) -> str:
    if isinstance(value, bytes):
        return value.decode("utf-8")
    return str(value)
```

from api.workflows.models import NodeResult
from api.workflows.start_chat.state import StartChatWorkflowState


def entry_node(state: StartChatWorkflowState, user_message: str) -> NodeResult:
    del user_message

def detect_intent(state: StartChatWorkflowState, user_message: str) -> str:
    profile_summary = state.profile_summary.lower()
    normalized_message = user_message.lower()

    if "wafer" in normalized_message and "이천" in profile_summary:

### 6. 시스템 프롬프트에 추가 문맥을 붙이는 예시

`api/llm/prompt/system.py`

```python
from datetime import datetime
from zoneinfo import ZoneInfo, ZoneInfoNotFoundError

from api import config

DEFAULT_SYSTEM_PROMPT = (
    "You are ITC OSS (Infra. Tech Center / One Stop Solution) Agent, Answer kindly in Korean. "
    "모르는 것은 모른다고 답하세요. 확실하지 않은 정보는 추측하지 마세요."
)


def get_system_prompt(*, user_profile_text: str = "") -> str:
    override = config.LLM_SYSTEM_PROMPT_OVERRIDE.strip()
    base_prompt = override if override else DEFAULT_SYSTEM_PROMPT

    sections = [base_prompt, _build_time_context()]
    profile_context = _build_profile_context(user_profile_text)
    if profile_context:
        sections.append(profile_context)
    return "\n\n".join(section for section in sections if section)


def _build_profile_context(user_profile_text: str) -> str:
    profile_text = user_profile_text.strip()
    if not profile_text:
        return ""

    return (
        "[사용자 프로필]\n"
        f"{profile_text}\n\n"
        "이 사용자의 소속과 근무 맥락을 고려해 적절한 workflow와 답변 방식을 선택하세요."
    )


def _build_time_context(now: datetime | None = None) -> str:
    current = now.astimezone(_get_llm_timezone()) if now else datetime.now(_get_llm_timezone())
    utc_offset = current.strftime("%z")
    formatted_offset = f"{utc_offset[:3]}:{utc_offset[3:]}" if len(utc_offset) == 5 else utc_offset
    return (
        "현재 한국 현지 시각은 "
        f"{current.strftime('%Y-%m-%d %H:%M:%S')} "
        f"({config.LLM_TIMEZONE}, UTC{formatted_offset})입니다. "
        "오늘, 내일, 이번 주 같은 상대적 시간 표현은 이 시각을 기준으로 해석하세요. "
        "사용자가 현재 시간이나 날짜를 물으면 이 시각 정보를 그대로 사용해 답하세요."
    )


def _get_llm_timezone() -> ZoneInfo:
    try:
        return ZoneInfo(config.LLM_TIMEZONE)
    except ZoneInfoNotFoundError:
        return ZoneInfo("Asia/Seoul")
```

## 선택 사항

상세 URL이 자주 느리거나 불안정하면 아래도 고려할 수 있다.

- 상세 URL 성공 결과를 Redis에 짧게 캐시
- `profile_source`를 `profile_api_cached`, `redis`, `unavailable`처럼 세분화
- 첫 턴에서는 fallback만 쓰고, 상세 프로필은 백그라운드 업데이트

현재 요구사항 기준으로는 "상세 URL 우선, 실패 시 Redis fallback, 실패해도 대화는 계속"만 먼저 넣는 것이 가장 현실적이다.

## 구현 순서 제안

1. 프로필 정규화 모델과 조회 서비스 추가
2. `start_chat` 상태에 프로필 필드 추가
3. `entry_node()`에서 선조회 후 상태 저장
4. `get_system_prompt()`와 `generate_reply()`에 추가 문맥 인자 연결
5. 테스트 추가
   - URL 성공 시 URL 결과 사용
   - URL timeout 시 Redis fallback 사용
   - 둘 다 실패해도 응답 생성은 계속
   - 시스템 프롬프트에 시간 + 프로필 문맥이 함께 들어감
