# Rich Notification Rules and Structure

## Overview
richnotificationis a JSON-based format for creating interactive bot messages with various UI components. 
This document outlines the rules and structure

---

## Root Structure

```json
{
    "richnotification": {
        "header": {...},
        "content": [...],
        "result": ""
    }
}
```

- All text arrays must have exactly 5 elements for multi-language support (It's OK to fill all with Korean as we normally use Korean)
- The `result` field at the root level is typically empty
- Empty strings are used for default values in my properties
- The `active` property is typically set to `true` for all controls
- Border styling is primarily used in grid layouts
- Image URLs should be accessible from the client's network
- Callback URLs should handle POST requests with form data

### Language Array Index (5 elements):
- Index 0 : Korean
- Index 1 : English
- Index 2 : Japanese
- Index 3 : Chinese
- Index 4 : Other/Empty


## 1. Header Section

### Required Feilds:
- `from`: Sender ID (e.g., "2067928", "X905552")
- `token`: Cube Bot Token
- `fromusername`: Array of bot names for differnt language (5 elements)
- `to`: Object containing recipeient information
  - `uniquename`: Array of recipeient unique names (sender ID)
  - `channelid`: Array of channel IDs
  - **note**: at least one of 'uniquename` or `channelid` is required

---

## 2. Content Section

### Structure:
```json
"content": [{
    "header": {},
    "body": {
        "bodystyle: "none" | "grid",
        "row: [...]
    },
    "process": {...}
}]
```
### bodystyle options:
- `none`: Default layout
- `grid`: Grid-based layout


## 3. Row and Column Structure

## Row properties:
- `bgcolor`: String - Border color (hex format)
- `border`: Boolean - Border presence (true/false)
- `align`: String - Left/right alignment ("left", "right", "center")
- `width`: String - Horizontal ratio (e.g., "100%")
- `column` : Array of column objects

### column properties:
- `bgcolor`: String - Background color (hex format)
- `border`: Boolean - Border presence (true/false)
- `align`: String - Left/right alignment ("left", "right", "center)
- `valign`: String - Top/bottom alignment ("top", "bottom", "middle")
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `type`: String - Rich type, type of screen control represented in column component ("label", "button", "checkbox", "select", "radio", "image", "hypertext", "inputtext")
- `control`: Control object with type-specific properties

---

## 4. Control Types

### 4.1 Label
**Purpose:** Display static text

**Properties:**
- `active`: Boolean
- `text`: Array of 5 strings (multi-language)
- `color`: Text color (hex format, e.g., "#000000", "#FF0000")

---

### 4.2 Button
**purpose: ** Clickable action button

**Properties:**
- `processid`: Process identifier (e.g., "Sentence", "SendButton", "SaveButton") - Event handling ID to distinguish user input components
- `active`: Boolean
- `text`: Array of 5 strings
- `confirmmsg`: String - Confirmation message displayed before event processing (e.g., "확인?")
- `value`: String - Event value, contains value information received from user or that can be received (e.g., "1", "첫번째")
- `bgcolor`: Background color (hex or empty)
- `textcolor`: Text color (hex or empty)
- `align`: Alignment ("center")
- `clickurl`: URL to open on click
- `androidurl`: Android-specific URL (normaly empty)
- `iosurl`: iOS-specific URL
- `popupoption`: Popup options
- `sso`: Boolean (SSO authentication) normally empty
- `inner`: Boolean

---

### 4.3 Radio
** Purpose:** Single selection option

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings
- `value`: Option value
- `checked`: Boolean (default selection)

**Note:** Multiple radio buttons with the same `processid` from a radio group (single selection).

---

### 4.4 Checkbox
**Purpose:** multiple selection option

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings
- `value`: Option value
- `checked`: Boolean (default selection)

**Note:** Multiple checkboxes with the same `processid` allow multiple selections

---

### 4.5 InputText
**Purpose:** Single-line text input

**Properties:**
- `processid`: Process identifier - Event handling ID
- `active`: Boolean
- `text`: Array of 5 string (label) - Multi-language
- `value`: String - Event value, default input value
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `minlength`: Integer - Minimum input length
- `maxlength`: Integer - Maximum input length
- `placeholder`: Array of 5 strings - Hint value, multi-language
- `validmsg`: Array of 5 strings - Input value check message, displayed when input string length doesn't meet min/max conditions, multi-language

---

### 4.6 Textarea
**Purpose:** Multi-line text input

**Properties:**
- `processid`: Process identifier -Event handling ID
- `active`: Boolean
- `text`: Array of 5 strings (label) -Multi-language
- `value`: String - Event value, default input value
- `minlength`: Integer - Minimum input length
- `maxlength`: Integer - Maximum input length
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `height`: String - Input field height (e.g., "100%", "100px")
- `placeholder`: Array of 5 strings - Hint value, multi-language
- `validmsg`: Array of 5 strings - Input value check message, multi-language

---

**Properties:**
