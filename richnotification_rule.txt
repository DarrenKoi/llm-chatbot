# Rich Notification Rules and Structure

## Overview
richnotification is a JSON-based format for creating interactive bot messages with various UI components.
This document outlines the rules and structure.

---

## Root Structure

```json
{
    "richnotification": {
        "header": {...},
        "content": [...],
        "result": ""
    }
}
```

- All text arrays must have exactly 5 elements for multi-language support (It's OK to fill all with Korean as we normally use Korean)
- The `result` field at the root level is typically empty
- Empty strings are used for default values in many properties
- The `active` property is typically set to `true` for all controls
- Border styling is primarily used in grid layouts
- Image URLs should be accessible from the client's network
- Callback URLs should handle POST requests with form data

### Language Array Index (5 elements):
- Index 0 : Korean
- Index 1 : English
- Index 2 : Japanese
- Index 3 : Chinese
- Index 4 : Other/Empty


## 1. Header Section

### Required Fields:
- `from`: Sender ID (e.g., "2067928", "X905552")
- `token`: Cube Bot Token
- `fromusername`: Array of bot names for different language (5 elements)
- `to`: Object containing recipient information
  - `uniquename`: Array of recipient unique names (sender ID)
  - `channelid`: Array of channel IDs
  - **note**: at least one of 'uniquename` or `channelid` is required

---

## 2. Content Section

### Structure:
```json
"content": [{
    "header": {},
    "body": {
        "bodystyle": "none" | "grid",
        "row": [...]
    },
    "process": {...}
}]
```
### content[].header
- Typically an empty object `{}`
- Pass `{}` as default

### bodystyle options:
- `none`: Default layout
- `grid`: Grid-based layout


## 3. Row and Column Structure

## Row properties:
- `bgcolor`: String - Border color (hex format)
- `border`: Boolean - Border presence (true/false)
- `align`: String - Left/right alignment ("left", "right", "center")
- `width`: String - Horizontal ratio (e.g., "100%")
- `column` : Array of column objects

### column properties:
- `bgcolor`: String - Background color (hex format)
- `border`: Boolean - Border presence (true/false)
- `align`: String - Left/right alignment ("left", "right", "center")
- `valign`: String - Top/bottom alignment ("top", "bottom", "middle")
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `type`: String - Rich type, type of screen control represented in column component ("label", "button", "checkbox", "select", "radio", "image", "hypertext", "inputtext", "textarea", "datepicker", "datetimepicker", "list", "container")
- `control`: Control object with type-specific properties

---

## 4. Control Types

### 4.1 Label
**Purpose:** Display static text

**Properties:**
- `active`: Boolean
- `text`: Array of 5 strings (multi-language)
- `color`: Text color (hex format, e.g., "#000000", "#FF0000")

---

### 4.2 Button
**Purpose:** Clickable action button

**Properties:**
- `processid`: Process identifier (e.g., "Sentence", "SendButton", "SaveButton") - Event handling ID to distinguish user input components
- `active`: Boolean
- `text`: Array of 5 strings
- `confirmmsg`: String - Confirmation message displayed before event processing (e.g., "확인?")
- `value`: String - Event value, contains value information received from user or that can be received (e.g., "1", "첫번째")
- `bgcolor`: Background color (hex or empty)
- `textcolor`: Text color (hex or empty)
- `align`: Alignment ("center")
- `clickurl`: URL to open on click
- `androidurl`: Android-specific URL (normally empty)
- `iosurl`: iOS-specific URL
- `popupoption`: Popup options
- `sso`: Boolean (SSO authentication) normally empty
- `inner`: Boolean

---

### 4.3 Radio
**Purpose:** Single selection option

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings
- `value`: Option value
- `checked`: Boolean (default selection)

**Note:** Multiple radio buttons with the same `processid` form a radio group (single selection).

---

### 4.4 Checkbox
**Purpose:** multiple selection option

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings
- `value`: Option value
- `checked`: Boolean (default selection)

**Note:** Multiple checkboxes with the same `processid` allow multiple selections

---

### 4.5 InputText
**Purpose:** Single-line text input

**Properties:**
- `processid`: Process identifier - Event handling ID
- `active`: Boolean
- `text`: Array of 5 strings (label) - Multi-language
- `value`: String - Event value, default input value
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `minlength`: Integer - Minimum input length
- `maxlength`: Integer - Maximum input length
- `placeholder`: Array of 5 strings - Hint value, multi-language
- `validmsg`: Array of 5 strings - Input value check message, displayed when input string length doesn't meet min/max conditions, multi-language

---

### 4.6 Textarea
**Purpose:** Multi-line text input

**Properties:**
- `processid`: Process identifier -Event handling ID
- `active`: Boolean
- `text`: Array of 5 strings (label) -Multi-language
- `value`: String - Event value, default input value
- `minlength`: Integer - Minimum input length
- `maxlength`: Integer - Maximum input length
- `width`: String - Horizontal ratio (e.g., "100%", "100px")
- `height`: String - Input field height (e.g., "100%", "100px")
- `placeholder`: Array of 5 strings - Hint value, multi-language
- `validmsg`: Array of 5 strings - Input value check message, multi-language

---

### 4.7 Select
**Purpose:** Dropdown selection

**Properties:**
- `processid`: Process identifier - Event handling ID
- `active`: Boolean
- `text`: Array of 5 strings (label) - Multi-language
- `item`: Array of option objects
  - `text`: Array of 5 strings - Item content, multi-language
  - `value`: String - Event value
  - `selected`: Boolean - Selection status

---

### 4.8 DatePicker
**Purpose:** Date selection

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings (label)
- `value`: Default date (format: "YYYY/MM/DD")

---

### 4.9 DateTimePicker
**Purpose:** Date and time selection

**Properties:**
- `processid`: Process identifier
- `active`: Boolean
- `text`: Array of 5 strings (label)
- `value`: Default date and time (format: "YYYY/MM/DD HH:MM")

---

### 4.10 Image
**Purpose:** Display image

**Properties:**
- `active`: Boolean
- `text`: Array of 5 strings (alt text / description)
- `linkurl`: URL to open on click - Web browser URL to popup when image is clicked
- `androidurl`: Android-specific URL - App URL to connect when button/image/hypertext is clicked in Android mobile environment
- `iosurl`: iOS-specific URL - App URL to connect when button/image/hypertext is clicked in iOS mobile environment
- `popupoption`: String - Popup options when opening linkurl or clickurl via browser
- `location`: Boolean - Whether image storage location is inside DMZ (true for internal)
- `sourceurl`: String - Image URL (e.g., "10.158.122.138/Resource/Image/test.png")
- `displaytype`: String - Display type for image on screen ("none": 기본 표시, "crop": 잘라서 표시, "resize": 자동 사이즈 조정)
- `width`: Image width (percentage or empty)
- `height`: Image height (empty or percentage)
- `sso`: Boolean - SSO integration required for event URL processing
- `inner`: Boolean - Whether target URL is inside DMZ for clickable screen components

---

### 4.11 List
**Purpose:** Display scrollable list of items

**Properties:**
- `active`: Boolean
- `text`: Array of 5 strings - Multi-language
- `pagesize`: Integer - Page size (0 for no pagination)
- `listitem`: Array of list item objects
  - `type`: String - Rich type ("label", "button", "checkbox", "select", "radio", "image", "hypertext", "inputtext")
  - `control`: Control object - Control content to put in list item

---

### 4.12 Container
**Purpose:** Group multiple controls together (used within list items)

**Properties:**
- `containeritem`: Array of container item objects
  - `align`: String - Alignment ("top", "middle", "bottom")
  - `width`: String - Width (e.g., "100%", "100px")
  - `itemcolumn`: Array of column objects - Column content to put in container (same structure as row columns)

---

### 4.13 HyperText
**Purpose:** Clickable text link

**Properties:**
- `active`: Boolean
- `text`: Array of 5 strings (link text)
- `linkurl`: String - Link address
- `androidurl`: String - Android-specific URL for mobile environment
- `iosurl`: String - iOS-specific URL for mobile environment
- `popupoption`: String - Popup options when opening linkurl or clickurl via browser
- `sso`: Boolean - SSO integration required for event URL processing
- `inner`: Boolean - Whether target URL is inside DMZ for clickable screen components
- `opengraph`: Boolean - Open Graph usage (required)

---

## 5. Process Section

**Properties:**
- `callbacktype`: String - Values: "url" (for HTTP callback)
- `callbackaddress`: String - URL to receive form submission (e.g., "http://10.158.121.214:17614/chatbot/chat/cube"). Can be empty for read-only notifications
- `processdata`: String - Additional process data (usually empty)
- `processtype`: String - Process type (usually empty)
- `summary`: Array of 5 strings - Summary text for each language (usually empty)
- `session`: Object
  - `sessionid`: Session identifier (e.g., "Bot_HR_0001")
  - `sequence`: Sequence number (e.g., "1")
- `mandatory`: Array of objects - Required field validation rules
  - `processid`: Process identifier to validate
  - `alertmsg`: Array of 5 alert messages
- `requestid`: Array of strings - List of process IDs to include in the callback
  - Common values: Control process IDs (e.g., "Sentence", "SelectDate", "Comment")
  - System IDs: "cubeuniquename", "cubechannelid", "cubeaccountid", "cubelanguagetype", "cubemessageid"

---

## 6. Data Type Rules

### Boolean Values
- Use lowercase: `true` or `false`

### Numeric Values
- In strings: "1", "2", "0"
- In numbers: `true`, `false` (for border)

### Color Values
- Hex format: "#ffffff", "#000000", "#FF0000", "#c4c4c4", "#d7d7d7"
- Empty string for default: ""

### Date Format
- Date: "YYYY/MM/DD" (e.g., "2014/12/08", "2018/08/23")
- DateTime: "YYYY/MM/DD HH:MM" (e.g., "2018/09/20 18:30")

### URL Format
- Must include protocol: `http://` or `https://`
- Example: "http://10.158.121.214:17614/chatbot/chat/cube", "https://www.skhystec.com/"

### Array Lengths
- Language arrays: Always 5 elements (Korean, English, Japanese, Chinese, Other)
- Empty arrays: `[]` or `["", "", "", "", ""]`

---

## 7. Validation Rules

### Input Validation
- `minlength`: Minimum character count (can be -1 for no limit)
- `maxlength`: Maximum character count (can be -1 for no limit)
- `validmsg`: Validation error message (5 languages)

### Mandatory Fields
- Use `mandatory` array in process section
- Each mandatory field has:
  - `processid`: The control’s processid
  - `alertmsg`: Alert message when field is empty (5 languages)

---

## 8. Layout Rules

### Column Width
- Full width: "100%"
- Half width: "50%"
- Quarter width: "25%"
- Empty string: Auto width

### Alignment
- `align`: "left", "center", "right"
- `valign`: "middle", "top", "bottom"

### Border
- `border`: `true` or `false`
- Used for table-like layouts

### Background Color
- `bgcolor`: Hex color or empty string
- Can be different for rows and columns

---

## 9. Process ID Naming Conventions

### Common Process IDs
- `Sentence`: General input/selection
- `Sentence1`, `Sentence2`: Multiple inputs
- `SendButton`: Submit action
- `SaveButton`: Save action
- `SelectContent`: Dropdown selection
- `SelectDate`: Date selection
- `SelectDateTime`: DateTime selection
- `SelectIcon`: Icon/image selection
- `SelectGiftSet`: Gift set selection
- `Evaluation`: Rating/evaluation
- `Idea`: Idea submission
- `Comment`: Comment input
- `CompleteDate`: Completion date
- `ProgressRate`: Progress rate
- `Reason`: Rejection reason
- `AgreeButton`: Approve action
- `RejectButton`: Reject action
- `Survey1`: Survey question

### System Process IDs (in requestid)
- `cubeuniquename`: User unique name
- `cubechannelid`: Channel ID
- `cubeaccountid`: Account ID
- `cubelanguagetype`: Language type
- `cubemessageid`: Message ID

---

## 10. Best Practices
- Always provide 5 language strings for text arrays, even if some are empty
- Keep consistent `processid` naming across controls
- Use predefined Process IDs when possible
- Ensure callback URLs are reachable from client environment
- Validate mandatory fields before submission
- Use empty string ("") as default instead of null
- Keep layout consistent using column width rules
- Separate UI structure (layout) and data (process values)
- Avoid hardcoding values inside UI definitions
- Use grid layout for complex forms with table-like structure

---
